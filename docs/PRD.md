# 產品需求文件（PRD）

**專案名稱：** 讀書筆記本系統  
**文件版本：** v1.0  
**建立日期：** 2026-04-23  

---

## 1. 專案概述

### 1.1 背景與動機

學生在學習過程中閱讀大量書籍與教材，但缺乏一個統一的平台來整理閱讀紀錄與心得。
本系統旨在提供一個簡單易用的讀書筆記管理工具，讓學生能夠依科目分類記錄讀書心得、評分，並快速搜尋過去的閱讀紀錄。

### 1.2 目標用戶

- **主要用戶：** 學生（中學生、大學生）
- **使用情境：** 課後複習、讀書心得整理、準備考試回顧閱讀紀錄

### 1.3 核心價值主張

> 讓學生能夠以「科目」為維度，輕鬆記錄、管理與搜尋每本書的閱讀心得與評分，建立屬於自己的數位讀書筆記本。

## 2. 功能需求

### 2.1 主要功能列表

| 功能編號 | 功能名稱         | 說明                                               |
|--------|----------------|--------------------------------------------------|
| F-01   | 科目管理         | 建立、編輯、刪除讀書科目（如：國文、英文、歷史）              |
| F-02   | 書籍紀錄管理      | 在科目下新增書籍，記錄書名、作者、閱讀日期、評分與心得         |
| F-03   | 讀後心得撰寫      | 針對每本書撰寫詳細的讀後心得，支援多段落文字輸入              |
| F-04   | 書籍評分         | 以 1～5 星為每本書評分，並在列表中顯示平均分               |
| F-05   | 書籍搜尋         | 以書名或關鍵字搜尋已記錄的書籍，快速找到目標筆記              |
| F-06   | 科目書籍列表      | 列出某科目下所有已記錄的書籍，按時間或評分排序               |
| F-07   | 首頁總覽         | 顯示所有科目及每個科目的書籍數量，提供快速入口               |

### 2.2 使用者故事

**F-01 科目管理**
> 作為學生，我希望能夠新增、編輯及刪除讀書科目，以便依課程分類管理我的讀書筆記。

**F-02 書籍紀錄管理**
> 作為學生，我希望能夠在各科目下新增書籍，記錄書名、作者與閱讀日期，以便保存完整的閱讀紀錄。

**F-03 讀後心得撰寫**
> 作為學生，我希望能夠為每本書撰寫詳細的讀後心得，以便日後複習時能快速掌握書本重點。

**F-04 書籍評分**
> 作為學生，我希望能夠對每本書給予 1～5 顆星的評分，以便反映我對這本書的喜好與推薦程度。

**F-05 書籍搜尋**
> 作為學生，我希望能夠透過書名或關鍵字搜尋書籍，以便在筆記數量龐大時快速找到特定書目。

**F-06 科目書籍列表**
> 作為學生，我希望能夠查看某個科目下所有書籍的列表，以便一覽無遺地掌握該科目的閱讀進度。

**F-07 首頁總覽**
> 作為學生，我希望在首頁看到所有科目與書籍數量的概覽，以便了解整體的讀書狀況。

## 3. 非功能需求

### 3.1 技術限制

| 項目       | 規格                            |
|----------|---------------------------------|
| 後端框架   | Python Flask                   |
| 模板引擎   | Jinja2                         |
| 資料庫     | SQLite                         |
| 前端       | HTML + CSS（Vanilla）+ JavaScript |
| 部署環境   | 本地端開發環境（localhost）        |

### 3.2 效能考量

- 頁面載入時間應在 2 秒以內（本地環境）
- 搜尋功能應即時回應，不需頁面完整刷新（可使用表單送出或 AJAX）
- SQLite 適用於單人或小型團隊使用情境

### 3.3 安全考量

- 表單輸入需進行基本驗證（書名不可為空）
- 防止 SQL Injection（使用 SQLAlchemy ORM 或參數化查詢）
- 本系統為單人使用，暫不實作登入驗證機制（Nice to Have）
